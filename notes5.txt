##Storage Management

start -----> boot process --------> login screen
1.power on :
BIOS(redhat) OR UEFI firmware is executed //
POST (Power On Self Test) is performed: 
   -Checks hardware components
   -Initializes hardware
   -Searches for bootloader
  
2. Bootloader (GRUB) is loaded :
      -GRUB (Grand Unified Bootloader) is a common bootloader used in Linux systems
      -GRUB is responsible for loading the Linux kernel and initial RAM disk (initrd) into memory
      -GRUB provides a menu to select the operating system or kernel version to boot

3.kERNEL is loaded :
      -The Linux kernel is loaded into memory by GRUB
      -The kernel initializes hardware, mounts the root filesystem, and starts the init process

4 Initramfs(Initial RAM Filesystem) process is started :
      -The init process is the first process started by the kernel
      -It is responsible for starting all other processes and services on the system
      -In modern Linux systems, systemd is commonly used as the init system

5. Switch to real root filesystem :
      -The initramfs process mounts the real root filesystem and switches to it
      -The system is now fully booted and ready for use

6.Init system starts :
      -The init system (e.g., systemd) starts various services and processes based on the configuration files
      -The system is now fully operational and ready for user interaction
      - older uses- SysVinit, Upstart, OpenRC, etc.

7. Syste, services and targets
      -The init system manages the startup and shutdown of services and targets
      -Services are individual programs that provide specific functionality (e.g., networking, web server)
      -Targets are groups of services that are started together to achieve a specific state (e.g., multi-user.target for a multi-user environment)

8. Login screen is displayed :
      -After all necessary services and targets are started, the login screen is displayed
      -Users can log in to the system and start using itLogout and shutdown :
      -Users can log out of the system, which will return them to the login screen
      -Users can also shut down or reboot the system using appropriate commands (e.g., shutdown, reboot)

##Creating partitions on a disk:
1. Identify the disk: Use commands like `lsblk` or `fdisk -l
2. Create a new partition: Use a partitioning tool like `fdisk`, `parted`, or `gparted` to create a new partition on the disk.
3. Format the partition: After creating the partition, format it with a filesystem (e.g., ext4, xfs) using the `mkfs` command.
4. Mount the partition: Create a mount point (e.g., /mnt/mydisk) and mount the partition using the `mount` command.
5. Update /etc/fstab: To ensure the partition is mounted automatically at boot, add an entry to the /etc/fstab file with the appropriate options.

##Mounting and unmounting filesystems:
1. Mounting a filesystem:
      - Use the `mount` command to mount a filesystem to a specific directory (mount point
      - Example: `mount /dev/sdb1 /mnt/mydisk` mounts the partition /dev/sdb1 to the directory /mnt/mydisk
2. Unmounting a filesystem:
      - Use the `umount` command to unmount a filesystem from its mount point
      - Example: `umount /mnt/mydisk` unmounts the filesystem from the directory /mnt/mydisk
3. Check mounted filesystems:
      - Use the `df -h` command to check the currently mounted filesystems and their usage
      - Use the `mount` command without arguments to see all currently mounted filesystems
4. Automatic mounting at boot:      
      - To ensure a filesystem is mounted automatically at boot, add an entry to the /etc/fstab file with the appropriate options
      - Example entry: `/dev/sdb1 /mnt/mydisk ext4 defaults 0 0` mounts /dev/sdb1 to /mnt/mydisk with default options at boot

      practical of RHEL9:
      $mount /dev/sdb1 /mnt/mydisk
      $umount /mnt/mydisk
      $df -h
      $mount
      $echo "/dev/sdb1 /mnt/mydisk ext4 defaults 0 0" >> /etc/fstab           

 ####Swap Partions:
   //when physical memory (RAM) is full, the system can use swap space on the disk to temporarily store data 
   that would otherwise be held in RAM. This helps prevent out-of-memory errors and allows the system to continue 
   functioning even when RAM is fully utilized.   
//processes in running state are stored in RAM, but when RAM is full, the system can move some of these processes to swap space on the disk. 
This allows the system to free up RAM for other processes and continue running without crashing. However, using swap space is slower than using
 RAM, so it is generally recommended to have enough physical memory to avoid excessive swapping.
