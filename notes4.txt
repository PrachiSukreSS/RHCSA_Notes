   ## Links : Soft/symbolic Links 
   link: ln -s target link_name
   hard link: ln target link_name

//diff between soft link and hard link?
hard link = 
-for file only
-same inode number
-same data block
-if original file is deleted, hard link will still work //resson - same inode number
-cannot cross filesystem
-cannot link to directory
-cannot link to non-existent file

Soft link:
-for file and directory
-different inode number
-different data block
-if original file is deleted, soft link will not work //reason - different inode number
-can cross filesystem
-can link to directory
-can link to non-existent file

##Process in OS 
#Types of process
1.User process - created by user
2.System process - created by OS
3.Daemon process - background process, created by OS or user
4.Interactive process - created by user, interact with user
5.Non-interactive process - created by user, do not interact with user
-kernel - 
//states of process:
ready, running, waiting, stopped, zombie

defucnt/zombie process - a process that has completed execution but still has an entry in the process table. It is waiting for its parent process to read its exit status. Once the parent process reads the exit status, the zombie process is removed from the process table.
check process state : 
>ps 
objects : -a - all process
           -u - user process
           -x - process without terminal
           -e - all process
           -f - full format
           ps -p <pid> - to check specific process
*** ->    ps -aux - to check all process with user and cpu and memory usage
         > kill all <service_name> - to kill all process of specific service
         > kill <pid> - to kill specific process
         > kill -9 <pid> - to force kill specific process
         > pkill <process_name> - to kill process by name
         > killall <process_name> - to kill all process by name
         > top - to check real time process and resource usage
         > htop - to check real time process and resource usage with interactive interface
         > nice - to set priority of process
         > renice - to change priority of running process

##Tar command - to archive and compress file
tar -cvf file.tar /path/to/directory //to archive file
tar -xvf file.tar //to extract file
tar -czvf file.tar.gz /path/to/directory //to archive and compress file
tar -xzvf file.tar.gz //to extract archived and compressed file
gzip -c file.tar > file.tar.gz //to compress file
bzip2 -c file.tar > file.tar.bz2 //to compress file
diff between gzip and bzip/bzip2:
gzip - faster compression and decompression, less compression ratio
bzip2 - slower compression and decompression, higher compression ratio

to update tar file:
tar -rvf file.tar /path/to/file //to add file to tar archive
tar -uvf file.tar /path/to/file //to update file in tar archive   

to remove file from tar archive:
tar -r  file.tar /path/to/file //to remove file from tar archive
