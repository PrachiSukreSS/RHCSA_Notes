#******compulsory 1 question#crontab -> schedule job to run at specific time
syntax:
* * * * * command to execute
- - - - -
| | | | |
| | | | +----- day of week (0 - 7) (Sunday=0 or 7)
| | | +------- month (1 - 12)
| | +--------- day of month (1 - 31)
| +----------- hour (0 - 23)
+------------- minute (0 - 59)                        
example:
0 5 * * * /home/user/backup.sh
This will run the backup.sh script every day at 5:00 AM.
prerequisite: cron service should be running
to check cron service status: systemctl status cron
to start cron service: systemctl start cron
to enable cron service at boot: systemctl enable cron
to edit crontab: crontab -e
to view crontab: crontab -l //list jobs of user
to remove crontab: crontab -r 
syntax:
from particular user: crontab -u username -e

package: yum install cronie
to check cronie status: systemctl status crond
to start cronie: systemctl start crond
to enable cronie at boot: systemctl enable crond
restart cronie: systemctl restart crond

SGID (Set group ID) - when we set sgid permission on a directory, the files created inside that directory will inherit the group ownership of the directory instead of the user who created the file.
to set sgid permission: chmod g+s directory_name
to check sgid permission: ls -l directory_name
example:
mkdir /data
chmod g+s /data //set sgid permission on /data directory WHERE s = sgid bit and g = group
ls -ld /data
drwxr-sr-x 2 root root 4096 Apr 26 12:00 /data //here s=sgid bit is set and group ownership is root 
when a user creates a file inside /data directory, the group ownership of the file will be root instead of the user's primary group.

Create /xdata with group inheritance
mkdir /xdata
chown :linuxadmin /xdata
chmod 2777 /xdata
💡 Why 2777?
2 → SGID bit
Files inside inherit group = linuxadmin
